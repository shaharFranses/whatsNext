# Backend Architecture Plan (FastAPI)

The backend must evolve from a simple script that asks Steam for data into a secure API that handles user sessions, coordinates multiple game store providers (Steam, GOG, Epic), aggregates a massive merged library, and protects against rate limits.

## High-Level Architecture

### 1. Authentication Layer (Supabase Auth)
- FastAPI must verify the Supabase JWT token sent securely from the Next.js frontend on every request to `/api/analyze` or `/api/library`.
- We will add a `get_current_user` dependency in FastAPI that extracts the user's UUID from the JWT, ensuring users can only fetch recommendations for their own linked accounts.

### 2. Provider Abstraction
Right now we have `steam.py`. We need to scale to `gog.py` and `epic.py`.
We will create a `BaseStoreProvider` class that defines a strict interface:
```python
class BaseStoreProvider(ABC):
    @abstractmethod
    async def get_owned_games(self, account_identifier: str) -> List[GameRecord]:
        pass
```
Each provider (Steam, GOG, Epic) will implement this and return a normalized `GameRecord` so the aggregator doesn't have to care where the game came from.

### 3. The Sync Engine 
When a user clicks "Analyze", doing live calls to 3 different stores takes too long.
Instead, we will add a background sync worker or an explicit `/api/sync` route:
1. The user connects GOG.
2. The frontend triggers a background sync.
3. FastAPI fetches the full GOG library and saves it into the Supabase `cached_library` table.
The `/api/analyze` endpoint will now read directly from `cached_library` instead of making live API calls to Steam/GOG/Epic, dropping response time from seconds to milliseconds.

### 4. Aggregator & Recommender Upgrades
**Multi-Source Merging:**
The Aggregator will now pull the top completed games (our Achievement DNA logic) across ALL providers. If a user has 100% completion in *The Witcher 3* on GOG and 100% in *Skyrim* on Steam, the Aggregator merges them seamlessly.

**Negative Tagging:**
The Recommender will respect user preferences stored in Supabase. If the user marks "Visual Novel" as a disliked genre, the Recommender will dynamically inject `& genres != (ID)` into the Apicalypse query generated by `igdb.py`.

## Core API Endpoints
- `POST /api/auth/connect/{provider}`: Save the user's Steam ID or Store OAuth token to Supabase.
- `POST /api/library/sync`: Trigger a background job to fetch libraries from all connected stores and write them to `cached_library`.
- `GET /api/analyze`: Calculate DNA and generate recommendations using the combined cached libraries and saved user dislikes.
- `DELETE /api/auth/disconnect/{provider}`: Remove a linked store account.
